# Vision-Language Learning Journey

This repository contains a 7-week learning path focused on **Computer Vision, Vision Transformers, and Vision-Language Models (VLMs)**.  
Each week includes conceptual explanations and hands-on experiments implemented in **Google Colab notebooks**.

---

##  Roadmap

### 1 — The Anatomy of an Image (CNN & Feature Maps)
Understanding what an image means for a machine.  
Introduction to classical computer vision concepts such as **Tensors, Convolution, Pooling, and Feature Maps**.



---

### 2 — The Transformer Revolution (Vision Transformer - ViT)
Moving beyond classical filters.  
Understanding how **Vision Transformers** process images as sequences of **patches**, including the **[CLS] token** and **Self-Attention mechanism**.



---

### 3 — Speaking the Same Language (CLIP)
How images and text meet in a shared mathematical space.  
Concepts include **Cosine Similarity**, **Projection Heads**, and **Contrastive Learning (InfoNCE Loss)** for aligning multimodal representations.


---

### 4 — Zero-Shot Learning & Prompt Engineering
Predicting unseen classes using only text descriptions.  
Exploring **Prompt Engineering**, **Few-Shot learning**, and model limitations such as **Hard Negatives**.


---

### 5 — From Understanding to Generation (Image Captioning)
Transitioning from matching tasks to text generation.  
Focus on **Encoder–Decoder architectures**, **Cross-Attention**, and building an **automatic product description generator** using **BLIP**.


---

### 6 — Auto-Labeling with GroundingDINO
Comparing traditional **supervised object detection (YOLO)** with **Zero-Shot VLM-based detection**.  
Using vision-language models to **reduce data labeling costs** and exploring **monocular distance estimation**.



---

### 7 — Capstone Project (End-to-End System)
Combining everything into a real-world pipeline:

**Smart Vehicle Recognition & Authorization System**

Pipeline components:
- **YOLO** → Vehicle & license plate detection  
- **EasyOCR** → Plate text extraction  
- **BLIP** → Image understanding  
- **Rule Engine** → Access authorization logic
